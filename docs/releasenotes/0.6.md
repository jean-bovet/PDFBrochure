<h2>PDFBrochure 0.6</h2>

<p>Polish update — the app looks better, behaves more deliberately, and is now backed by an automated test suite.</p>

<ul>
    <li><b>Green book icon.</b> Replaces the previous black-on-white glyph.</li>
    <li><b>Result screen after creating.</b> Clicking <b>Create Brochure</b> now switches to a dedicated screen that shows a spinner while the PDF is generated, then offers <b>Reveal in Finder</b> or <b>Make another brochure</b>. On error, the screen offers <b>Try again</b> (keeping your loaded PDF and settings) or <b>Start over</b>.</li>
    <li><b>Cleaner drop screen.</b> The redundant title is gone; the call-to-action <i>"Drop a PDF to make a brochure"</i> now lives directly inside the drop zone.</li>
    <li><b>Smarter paper-size detection.</b> US Letter input no longer gets mistakenly treated as A4. The previous heuristic compared total area, which placed Letter and A4 within 3% of each other; the new check compares both width and height, so each format routes to its native sheet size.</li>
</ul>

<p>Behind the scenes: the SwiftUI source was split into one-file-per-screen with <code>#Preview</code> blocks, and a 35-test suite now covers both the saddle-stitch imposition algorithm (verified by rendering output sheets and pixel-sampling each slot) and the new drop / layout / result state machine.</p>

<p><i>Apple Silicon only. Requires macOS 13 or later.</i></p>
